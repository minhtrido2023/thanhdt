State-conditional capital-budgeting engine.

**Design**: DT5G state → POLICY row of target NAV fractions {core, value, capit, grind}, rest = cash. ACTIVATION GATES: value OFF in CRISIS (gated); capit only on washout (oversold high); grind only in NEUTRAL grind (flat index + breadth<0.45). Inactive sleeve's budget → core, then capped → cash. CAPS: per-sleeve (value .35/capit .40/grind .20), max total sleeve .50, and **STATE-DEPENDENT core cap** {CRISIS 0.10, BEAR/EXBULL 0.50, NEUTRAL/BULL 0.70} — the key fix: in CRISIS momentum INVERTS (#3 fitness matrix) so core is capped to 0.10, forcing an unfired-capit budget to overflow to CASH not bad-regime momentum.
POLICY: CRISIS core.10/capit.40 (else cash); BEAR .40/.30/.10; NEUTRAL .45/.25/grind.15/capit.05; BULL .60/.35; EXBULL .50/.30 (trimmed, fragility #4).
