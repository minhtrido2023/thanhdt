BƯỚC 2 — Expanding Window Percentile Rank

2Tính rank từng yếu tố — không dùng dữ liệu tương lai

Expanding percentile rank tại thời điểm t: rank_f[t] = #{s ≤ t : f[s] ≤ f[t], f[s] không phải NaN} ───────────────────────────────────────────── #{s ≤ t : f[s] không phải NaN} Điều kiện: cần ít nhất min_lookback = 252 ngày hợp lệ Nếu chưa đủ 252 điểm → rank_f[t] = NaN (chưa tính) 
def expanding_pct_rank(series: np.ndarray, min_lb: int = 252) -> np.ndarray:
    """
    Tính expanding percentile rank — không look-ahead.
    Trả về giá trị trong [0, 1]; NaN nếu chưa đủ min_lb điểm.
    """
    out = np.full(len(series), np.nan)
    for t in range(len(series)):
        hist = series[:t+1]
        valid = hist[~np.isnan(hist)]
        if len(valid) < min_lb:
            continue                        # chưa đủ dữ liệu
        out[t] = np.sum(valid <= series[t]) / len(valid)
    return out
Tại sao expanding, không rolling? Rolling 252 ngày sẽ "quên" các giai đoạn cực đoan trong quá khứ. Expanding window giữ lại toàn bộ lịch sử — một đỉnh bull run 2007 vẫn ảnh hưởng đến ngưỡng phân loại năm 2022.

BƯỚC 3 — Composite Score

3Bình quân có trọng số của 7 rank yếu tố

Trọng số cơ sở: W = {P3M: 0.30, P1M: 0.10, MA200: 0.15, RSI: 0.15, MACD: 0.10, CMF: 0.08, Breadth: 0.12} Composite score tại t (chỉ dùng yếu tố có sẵn): available = {k : rank_k[t] không phải NaN} Yêu cầu: |available| ≥ 3 score[t] = Σ (rank_k[t] × W[k]) for k in available ───────────────────────────────────────── Σ W[k] for k in available (chuẩn hóa lại để tổng trọng số = 1) 
def composite_score(ranks: dict, W_BASE: dict, min_factors: int = 3) -> float:
    avail = {k: v for k, v in ranks.items() if not np.isnan(v)}
    if len(avail) < min_factors:
        return np.nan
    w_sum = sum(W_BASE[k] for k in avail)
    return sum(avail[k] * W_BASE[k] for k in avail) / w_sum

⚠️ Đây là bước quan trọng nhất và thường bị hiểu nhầm. Trạng thái thị trường được xác định bởi r_score — vị trí của composite score trong phân phối lịch sử của chính nó — không phải giá trị tuyệt đối của score. Một score = 0.62 có thể là CRISIS nếu 90% các ngày trước đó có score cao hơn (ví dụ: sau bull run 2020–2021).

BƯỚC 7 — Trailing Mode Smoothing (5 kỳ)

7Lọc nhiễu bằng rolling mode — chỉ dùng quá khứ

smoothed_state[t] = mode(state[t-4], state[t-3], state[t-2], state[t-1], state[t]) ← mode = trạng thái xuất hiện nhiều nhất trong 5 kỳ gần nhất ← nếu hòa (2-2-1): ưu tiên trạng thái gần nhất 
from scipy.stats import mode as scipy_mode

def rolling_mode(states: np.ndarray, window: int = 5) -> np.ndarray:
    out = states.copy()
    for t in range(window - 1, len(states)):
        window_vals = states[t-window+1 : t+1]
        m = scipy_mode(window_vals, keepdims=True)
        out[t] = int(m.mode[0])
    return out
Smoothing ngăn hệ thống flip-flop liên tục khi composite score dao động quanh ngưỡng. Nếu state đổi 3/5 ngày trở lên mới ghi nhận là chuyển đổi thực sự.

BƯỚC 8 — Thực thi danh mục

8T+1 execution delay + 3-day gradual ramp + chi phí giao dịch

Tham số: TARGET_W = {1: 0.00, 2: 0.20, 3: 0.70, 4: 1.00, 5: 1.30} DEPOSIT_R = 0.06 / 252 ← lãi suất tiết kiệm 6%/năm (phần tiền mặt) BORROW_R = 0.10 / 252 ← lãi suất margin 10%/năm (phần vượt 100%) TC = 0.001 ← phí giao dịch 0.1% mỗi lần mua/bán RAMP = 3 ← chuyển dần qua 3 ngày SNAP_THR = 0.03 ← nếu chênh < 3% → chuyển ngay lập tức Vòng lặp hàng ngày: w_prev ← trọng số cổ phiếu ngày hôm qua target = TARGET_W[ smoothed_state[t-1] ] ← T+1 delay diff = target - w_prev if |diff| < SNAP_THR: w_new = target ← snap nếu nhỏ else: w_new = w_prev + diff / RAMP ← tiến 1/3 mỗi ngày r[t] = Close[t] / Close[t-1] - 1 ← daily return VNINDEX cash_r = max(0, 1 - w_new) × DEPOSIT_R ← lãi tiền mặt margin_c = max(0, w_new - 1) × BORROW_R ← chi phí margin trade_c = |w_new - w_prev| × TC ← phí giao dịch pnl[t] = w_new × r[t] + cash_r - margin_c - trade_c PV[t] = PV[t-1] × (1 + pnl[t]) 
def backtest(close: np.ndarray, smoothed_state: np.ndarray) -> np.ndarray:
    TARGET_W   = {1:0.00, 2:0.20, 3:0.70, 4:1.00, 5:1.30}
    DEPOSIT_R  = 0.06 / 252
    BORROW_R   = 0.10 / 252
    TC         = 0.001
    RAMP       = 3
    SNAP_THR   = 0.03
    n          = len(close)
    pv         = np.zeros(n);  pv[0] = 1e9   # vốn ban đầu 1 tỷ VND
    w          = 0.70                          # trọng số khởi đầu = NEUTRAL

    for t in range(1, n):
        target = TARGET_W[smoothed_state[t-1]]   # T+1 delay
        diff   = target - w
        w_new  = target if abs(diff) < SNAP_THR else w + diff / RAMP
        r      = close[t] / close[t-1] - 1
        cash_r = max(0, 1 - w_new) * DEPOSIT_R
        marg_c = max(0, w_new - 1)  * BORROW_R
        trd_c  = abs(w_new - w)         * TC
        pv[t]  = pv[t-1] * (1 + w_new*r + cash_r - marg_c - trd_c)
        w      = w_new
    return pv
