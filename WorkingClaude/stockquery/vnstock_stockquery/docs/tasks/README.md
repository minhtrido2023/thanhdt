# Task Briefs

Use this directory for lightweight task briefs that should survive beyond chat history.

Good candidates:

- multi-file bug fixes
- user-reported behavior regressions
- focused feature requests that do not need a full execution plan
- handoff notes for unfinished work

Recommended filename:

- `YYYY-MM-DD-short-task-name.md`

Start from `../templates/TASK_BRIEF_TEMPLATE.md` when needed.
