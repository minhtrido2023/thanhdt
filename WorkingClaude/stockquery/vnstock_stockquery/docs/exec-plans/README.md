# Execution Plans

Use this directory for living plans when work is too large, risky, or cross-cutting for an informal brief.

## Layout

- `active/`: in-progress plans
- `completed/`: finished plans kept for repository memory

## Rules

- Plan format is defined in `../../PLANS.md`.
- Start from `../templates/EXEC_PLAN_TEMPLATE.md`.
- Keep the file current as implementation proceeds.
- Move finished plans from `active/` to `completed/` rather than deleting them.
